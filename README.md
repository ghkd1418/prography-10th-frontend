## 구현 스펙
1. 리크루팅 진행 단계 표시
- 현재 리크루팅 진행 단계를 시각적으로 표시합니다.
 <img width="412" alt="스크린샷 2025-02-21 오전 11 40 18" src="https://github.com/user-attachments/assets/d02c1d93-a372-46c8-800c-d977a5d8db4d" />

2. 폼 데이터 유지
- 사용자가 제출하기 버튼을 누르기 전까지 입력한 리크루팅 폼 데이터를 유지해야 합니다.
- useFunnel 훅을 활용하여 내부적으로 useState를 통해 폼 데이터를 저장하고 관리합니다.

3. 필수 입력 항목 검증
- 	필수 입력 항목을 설정하여, 입력되지 않을 경우 다음 단계로 이동할 수 없도록 제한합니다.
- react-hook-form의 required 옵션을 활용하여 필수 입력을 검증합니다.

4. 필수 입력 누락 시 경고 및 알림
- 필수 입력 항목이 누락되었을 경우, 경고 메시지 및 알림을 제공합니다.
  <img width="1303" alt="스크린샷 2025-02-21 오후 1 19 22" src="https://github.com/user-attachments/assets/7cd7662d-a54f-441e-a698-120a2fddfae2" />

5. 특정 항목 예시 데이터 표시
- 	특정 입력 필드에 대한 예시 데이터를 제공하여 사용자의 입력을 도와줍니다.
  <img width="1299" alt="스크린샷 2025-02-21 오후 1 19 36" src="https://github.com/user-attachments/assets/c78b32e7-62e7-44a8-83d3-869617a819b2" />

6. API 통신을 위한 폼 데이터 구성
- handlePostApi 함수에서 API 요청을 위한 리크루팅 폼 데이터를 구성하고 전송합니다.

7. 이 외에도 UI/UX적으로 필요한 기능이 있다면 **자유롭게 추가**해 주세요.
- 전역 에러 바운더리를 추가하여 예기치 않은 오류 발생 시 사용자 경험을 개선
- 반응형 UI 적용

 
 ### 🤔 use-funnel 커스텀 훅
	- 원래는 Toss의 use-funnel을 사용하려 했으나,
	 -  아직 patch 버전이고,
	 -  해당 프로젝트에서 필요하지 않은 기능이 많으며 구조가 복잡하다고 판단
	
 따라서 간단한 use-funnel을 직접 구현하여 사용했습니다.
